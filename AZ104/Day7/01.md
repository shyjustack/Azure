## Azure VM Security Types
| Security Type             | Description                                                                                                                                                                        | Typical Use Case                                                                                                            |
| ------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| **Standard**              | The default VM security type. Runs like a traditional VM with standard virtualization and no extra hardware-based protections.                                                     | Most general-purpose workloads, legacy OS support, or when you need compatibility.                                          |
| **Trusted Launch**        | Enhanced security type that protects against rootkits, bootkits, and firmware-level attacks. Enables **Secure Boot**, **vTPM**, and **Boot Integrity Monitoring**.                 | Recommended for production workloads that require higher trust or compliance (e.g., ISO, PCI, or Azure Security Benchmark). |
| **Confidential VM (CVM)** | Provides *memory encryption and isolation* from the Azure host/hypervisor using **AMD SEV-SNP** or **Intel TDX**. Even Azure administrators can’t access your VM’s memory or data. | Highly sensitive data, confidential workloads, finance, healthcare, government, or R&D workloads.                           |
